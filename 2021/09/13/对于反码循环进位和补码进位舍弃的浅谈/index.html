反码循环进位与补码的进位舍弃理解

相信大家在学计算机的时候，二进制接触最多的就是 原码，反码，补码，本篇博文将会简单介绍这三种 二进制编码，然后再谈鄙人对反码补码计算规则的理解。

对于三种编码的简单介绍 ：

    1. 原码：

      原码(true form)是一种计算机中对数字的二进制定点表示方法。 原码表示法在数值前面增加了一位符号位（即最高位为符号位）：正数该位为0，负数该位为1（0有两种表示：+0和-0），其余位表示数值的大小。

      相信聪明的读者肯定会想到，既然0的表示不唯一，那它肯定不能再计算机中使用，因为计算机不能像真人一样有思考，它只是运算的机器。这是一点。还有一点就是因为原码的计算总出现错误，举个简单的例子，0001（1） + 1001（-1） = 1010（-2），这显然是荒谬的。于是便出现了反码。

    2. 反码：

      反码是一种在计算机中数的机器码表示。 对于单个数值（二进制的0 和1）而言，对其进行取反操作就是将0 变为1，1 变为0。 正数的反码和原码一样，负数的反码就是在原码的基础上符号位保持不变，其他位取反。看到此处只能让你会求反码，但要理解还请往下看。在上个问题上 0001（1） + 1001（-1） = 1010（-2），用反码的话就会得到：0001（-1） + 1110（-1） = 1111（-0），虽然结果是-0，但是显然结果比直接错误要好。但是，又一想，为啥不是唯一的0。我们接着计算：1110（-1）+ 1101（-2） = 1011（-4）? ? ? 为何会这样？但是，似乎问题不大，因为对于两个负数相加，我们完全可以当成整数，然后把符号位强制换成1，但是0不唯一的问题还是没法解决。于是出现了补码，这个0唯一，且特征明显的好方法。

    3. 补码：

      补码的定义建议自己百度，因为它具有某些特殊含义。但是计算方法这里可以浅浅道来。

      1. 利用反码 + 1 获得
      2. 先求原码，从最低位往前看，从第一个1开始，后面每一位取反，符号位不变。当然，正数的补码还是和原码一样。

对于反码（循环进位）和补码（进位舍弃）计算规则的理解。

    1. 首先简单介绍一下反码：

      1. 正数的反码等于原码，等于本身，但是负数就不是，负数相当于在原数的基础上加了一个2^n-1，至于为啥是2^n - 1，且听我娓娓道来。

        首先，在值为负数的情况下，以4位二进制数为例，比如-3，它的反码是1100，如果按二进制的计算规则，1100就是12，这里你可能还有点

        是不是突然发现，他们的和都是 15 ？

        想必大家已经有一点感觉了，那就听我接着来谈。

        15是啥？1111？ -0？没错，就是
        $$

        如果是一面钟表的话，反码的“钟表”最后一格相当于没有内容，其他很类似，比如-1，那就往前走一个单位，是14，-0就是15，是不是感觉很奇妙？这就对了。同样的相应的整数，就是正常的往后走。

        于是我们可以得出来：
        $$
        x（反码） = x ，x是正数，
        x（反码） = x + 2^n - 1 ，x是负数
        $$
        此处的反码，均可理解为无符号数。

    2. 正餐开始，鄙人对于循环进位的理解：

      因为进位有溢出的情况，此处只讨论不溢出的问题。

      大家都知道，两个数相加有3种情况：

      • 正 + 正

      • 正 + 负

      • 负 + 负

      • 1 . 因为正数的反码与原码一样，所以正数 + 正数，答案一定正确，

        1. 当一正一负时：假设第一个数为A，第二个数为B

          1. A + B > 0

            此时，一定会产生进位

            接下来，我们要知道我们的目标，因为 A + B > 0 ,于是 （A + B）的反码是正数，

            就是 A（反码） + B （反码） = （A + B）反码 = （A+B）

            又 A(反码) + B（反码） = A + B + 2^n - 1，

            那么如何 把 A + B + 2^n - 1 —> A + B 呢？

            很简单，- 2^n + 1，那如何操作呢？就是 舍去进位的 1（由于进位是最高位，于是，1 就代表了2^n ），再在最后加上 1。

            至此，转换完成。

          2. A + B < 0

            此时一定不会产生进位

            于是此时，直接相加就是结果（此结果是反码）。

          3. 对于 A + B = 0,由于产生进位，在此，不进行讨论。

        2. 当 两个全为负的时候，假设第一个数为A，第二个数为B

          此时一定有进位：

          于是 和上面一样，我们要找我们的目标，来确定如何变换，让结果正确。

          很显然，我们的目标就是 (A + B) + 2^n - 1，

          但是似乎，如果单纯的 反码相加 多了一个 2^n - 1，

          因此，我们就和上面的操作一样，舍去最高进位，相当于减去 2^n 然后再在最后加上1，

          就变成我们想要的结果了。

          至此，完成。

      其实补码的进位舍弃和反码类似，只是将原来的 2^n - 1 变为 2^n。
